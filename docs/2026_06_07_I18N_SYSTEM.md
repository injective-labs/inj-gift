# InjGift 多语言（i18n）系统

> 2026-06-07 ：把分散在各页面的硬编码文案统一收敛到一套共享字典，彻底解决“切了语言但部分文字还是中文”的混用问题。

## 背景

改造前，多语言只存在于首页 `src/app/page.tsx`：

- `Locale` 类型、`copy`、`featurePanelCopy` 都定义在该文件内部；
- 语言选择是 `Home` 组件里的 `useState`，**不跨页面、不持久化**；
- 其他路由（`/create`、`/claim`、`/claim/[id]`、`/packet`、`/packet/[id]`）以及钱包组件、所有 toast / 校验提示，全部是写死的中文。

所以一旦切换语言或跳转页面，就出现中英（日韩）混用。

## 现在的结构

所有文案集中在 `src/i18n/`：

| 文件 | 职责 |
| --- | --- |
| `config.ts` | `Locale` 类型、`localeOrder`、`localeNames`（语言原生名称）、`localeHtmlLang`、默认语言、localStorage key、`isLocale` 守卫 |
| `messages.ts` | 唯一的文案字典。以 `zh` 为基准对象，`type Dict = typeof zh`，`en`/`ja`/`ko` 标注为 `Dict`，**缺 key 会编译报错** |
| `provider.tsx` | `I18nProvider`（持久化 + 同步 `<html lang>`）与 `useI18n()` hook |
| `errorMessage.ts` | `errorMessage(e, t)`：把抛出的错误解析为当前语言的提示 |
| `index.ts` | 统一出口 |

`I18nProvider` 在 `src/app/layout.tsx` 里包裹全应用，语言写入 `localStorage`（key：`injgift.locale`），刷新和跨页面都会保留。

## 字典分区

`messages` 每个语言下分为：`common`（通用原子词）、`form`（表单标签）、`status`（红包状态）、`home`（首页）、`panel`（首页内嵌的创建/领取/管理面板）、`create` / `claimIndex` / `claimDetail` / `packetIndex` / `packetDetail`（各独立路由）、`wallet`（钱包按钮/弹窗/钱包项 hint）、`errors`（所有 toast、校验、归一化错误文案）。

## 在组件里使用

```tsx
"use client";
import { useI18n } from "@/i18n";

export function Demo() {
  const { t, locale, setLocale } = useI18n();
  return <button>{t.common.copy}</button>;
}
```

- `t` 是「当前语言」的整棵字典（`messages[locale]`）。
- 切换语言：`setLocale("en")`。
- 列表/下拉用 `localeOrder` 遍历，显示名用 `localeNames[locale]`。

## 错误文案的本地化

`normalizeError` 仍返回原始 `message`（保持单测稳定、便于日志），但额外带上 `messageKey`（指向 `t.errors` 的稳定 key）。展示层用 `errorMessage(e, t)` 解析：

```tsx
import { errorMessage } from "@/i18n";
catch (e) { toast.error(errorMessage(e, dict) || dict.errors.claimFailed); }
```

`AppError.messageKey` 为可选字段（见 `src/domain/errors.ts`），新增不影响既有调用与测试。

## 新增文案的规范

1. **先在 `messages.ts` 的 `zh` 里加 key**，TypeScript 会提示 `en`/`ja`/`ko` 也要补齐。
2. 通用词放 `common`，表单标签放 `form`，错误/提示放 `errors`，页面专属放对应分区。
3. 组件里一律用 `t.xxx`，**不要写死字面量**。
4. 动态片段用拼接（如 `` `${t.errors.createSuccess}: ${hash}` ``），不要把变量塞进字典。

## 已知遗留

- `src/features/redpacket/domain/errors.ts`（`mapRedPacketError` / `FRIENDLY_MAP`）仍含中文，但该路径**未被任何页面引用**（死代码）。若将来启用 `features/redpacket/hooks/*`，应改为返回 error code，并在调用处用 `errorMessage` 翻译。
- `src/app/debug/page.tsx` 是开发调试页，文案为英文，未纳入 i18n。
